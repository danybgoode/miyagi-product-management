# SmallDocs report hub branding plan

**Status:** Stories 1-2 shipped live 2026-07-14 via SmallDocs PR #1
(`cea02aa9db690f0b2c39dd1748f901f2a178d195`) · Story 3 shipped live 2026-07-14 via SmallDocs PR #2
(`eee8803b784f0577d15227e29d0d56fff317f1a8`) · Story 4 planned · **Risk:** LOW for branding and
hosted public Roadmap projection, MED when true short-link storage is chosen · **Service:** `pmo-smalldocs`

The `pmo-smalldocs` root page is now the internal Miyagi Reports hub: a branded place for PMO, standup,
weekly recap, Roadmap navigation, and future stakeholder packets. The remaining open decision is true
short-link storage; today report payloads still travel in URL-hash links with short Telegram labels.

## Story 1 — Branded report-hub landing page ✅

**As** Daniel, **I want** `https://pmo-smalldocs-oehqqtyoia-uk.a.run.app/` to open as the Miyagi report
hub, **so that** shared report links feel like our operating system rather than upstream product
marketing.

**Acceptance:**
- Root `/` says Miyagi Reports / PMO Report Hub, not generic SmallDocs marketing.
- First screen has direct cards for Weekly PMO, Daily Standup, Monthly Packet, Metrics Sheet, and Trust.
- The page explains, briefly, that reports travel in URL-hash links unless a future short-link registry is
  enabled.
- `/trust` and license notices remain visible and intact.
- No third-party hosted service, no upstream notice removal, no license-key workaround.

## Story 2 — Miyagi report theme ✅

**As** a stakeholder opening a report, **I want** decks/docs/sheets to carry a Miyagi PMO visual identity,
**so that** the artifacts feel intentional when forwarded.

**Acceptance:**
- Favicon, document title, app chrome, default theme colors, and export metadata read as Miyagi/PMO.
- Story-deck templates honor `styles.slideAspectRatio: "16:9"` for PMO and standup decks.
- Empty/loading/error states use the same brand voice.
- Upstream SmallDocs trust/license surfaces remain available.

## Story 3 — Hosted Roadmap report library ✅

**As** Daniel, **I want** `/reports` to show curated Roadmap, epic, sprint, shipped, and idea-funnel
views, **so that** the report hub becomes a polished public navigation layer for clients, investors, and
the team.

**Acceptance:**
- `/reports` renders hosted data from `public/reports-data.json` with no local loopback dependency.
- The `/docs` toolbar button `Biblioteca de reportes` opens `/reports`, not `/library`.
- `/library` and `/connect` remain intact for the upstream local-agent/private-file flow.
- Every report and executive view opens through SmallDocs `/docs#md=...`.
- The generated data is curated from Roadmap projection fields and compact summaries, not a raw dump of
  every markdown detail.
- Desktop and phone layouts avoid horizontal overflow and keep filters usable on mobile.

## Story 4 — True short report links ⏳

**As** Daniel, **I want** Telegram links like `/r/pmo-weekly-2026-07-14` or `/r/daily-story-today`,
**so that** report links are short, readable, and resilient even when the embedded URL-hash payload gets
large.

**Acceptance:**
- A report registry maps a short slug to the immutable markdown payload or compressed hash.
- Slugs are generated by the report scripts with a predictable/friendly prefix plus collision-safe suffix.
- Telegram messages can use the real short URL, not just an HTML label hiding a long URL.
- Registry storage is explicit before build: Cloud Storage/Firestore/Supabase/etc. This is the risk fork.
- Reads are public/no-auth for forwarded reports; writes are restricted to the routine/service account.
- Retention is decided up front: keep forever for stakeholder packets, or TTL daily standups.

## Recommended order

1. **Shipped:** Telegram HTML labels (`abrir deck semanal`, `abrir daily story`) keep messages readable
   while preserving URL-hash sharing.
2. **Shipped:** SmallDocs fork PR #1 replaced the generic landing page with Miyagi Reports branding and
   localized the viewer chrome without changing storage.
3. **Shipped:** SmallDocs fork PR #2 added `/reports`, a hosted public Roadmap projection generated by the
   root repo and opened through the existing `/docs#md=...` viewer.
4. **Next decision:** enable a tiny report registry only after choosing retention,
   slug shape, and write credentials. This is the only part that changes the current stateless posture.
