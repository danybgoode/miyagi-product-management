# "Ask Claude about miyagisanchez.com" — outreach campaign (raw, pre-scope)

> RAW capture (2026-06-07). Creative direction only — the campaign itself is designed later, in a
> separate chat. Captured here so the #6 landing pages can carry the same spine.

## The core mechanic
Don't ask people to trust us — tell them to ask **their own** AI. The CTA *is* a prompt the reader
can paste into Claude / Gemini / ChatGPT. The agent does the convincing, grounded and tailored, as an
**independent auditor** with no incentive to oversell. Worst case: free distribution.

## Creative lines (es-MX, drafts to refine)
- *"¿No nos crees? Pregúntale a Claude: «hola, ¿me puedes decir qué es miyagisanchez.com?»"*
- *"¿tf is Bezos? Pregúntale a Claude sobre miyagisanchez.com."*
- *"No confíes en nosotros. Confía en tu propia IA."*
- *"Tu asistente ya te conoce. Pregúntale si Miyagi es para ti."*

## Surfaces (later)
Print (the México-86 magazine + flyers), digital ads, social, QR → the prompt. Variations per
persona and per channel.

## Hard dependency (why this is also a product task, not just creative)
The campaign only works if an agent that fetches `miyagisanchez.com` gets a **great, grounded,
supply-side answer**. Today's agent surface is buyer/catalog-oriented. → the **agent-readable
why-sell/about surface** (see README · Spawned ask #2) must ship for the campaign to deliver. The #6
landing pages must also be **agent-fetchable** (semantic HTML, real text, structured metadata).

## Measurement (later)
Attribution from "asked an agent" → signup is hard (the agent step is off-platform). Options to
explore at campaign design: prompt-embedded UTM links the agent surfaces, unique landing slugs per
creative, QR codes. Microsoft Clarity (connected) for on-site behavior.
